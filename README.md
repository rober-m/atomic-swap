# atomic-swap

A Cardano protocol project generated by [cardano-init](https://github.com/aspect-build/cardano-init).

## Protocol

A simple on-chain atomic swap:

- **Lock** — Person A locks some assets at the swap script address, attaching an
  inline datum that records A's payment key hash (`owner`) and the `price` A
  wants in return.
- **Accept** — Any Person B can consume that UTxO, taking the locked assets, as
  long as the transaction contains exactly one output that pays `owner` at least
  the `price` and carries the swap UTxO's own output reference as its datum. Both
  legs settle in one transaction or neither does.
- **Cancel** — A can reclaim the locked assets at any time by signing the
  transaction, so funds are never stuck if no counterparty appears.

Requiring the payment output to be marked with the consumed swap UTxO's output
reference binds that payment to one specific swap: since every UTxO's output
reference is globally unique, a single payment can never satisfy two swaps at
once (the "double satisfaction" attack), and requiring exactly one such output
forces the payment into a single UTxO. This uniqueness is guaranteed by the
ledger, so A never has to manage a nonce.

Off-chain CLI (`off-chain/`):

```bash
npx tsx src/cli.ts lock <lockLovelace> <askLovelace>  # A locks, asking a price
npx tsx src/cli.ts accept <lockTxHash>                # B accepts and pays A
npx tsx src/cli.ts cancel <lockTxHash>                # A reclaims
```

## Quick Start

```bash
just build    # Build all components (on-chain first, so the blueprint exists)
just test     # Build the blueprint, then run every component's tests in order
just clean    # Remove build artifacts
```

## Development

`build`, `test`, and `clean` run from the top level. Some components also expose
a long-running `dev` — a watch mode, a local devnet, or a REPL. Not every
component has one (it's optional), so check with `just -f <dir>/Justfile --list`
and run it directly, e.g. `just -f <dir>/Justfile dev`.

When a component provisions a local devnet, its `dev` writes the connection
details into `.env` and the off-chain component connects to it automatically;
with no local devnet it falls back to a public provider (Blockfrost). You never
wire this by hand — it follows the presence of `INDEXER_URL` in `.env`.
